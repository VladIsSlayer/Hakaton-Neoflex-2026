# Frontend Specification (React + TypeScript)

## Архитектура и Стилистика
- **Core:** React 18+ с типизацией на TypeScript. Основа для быстрой сборки — Vite (или Next.js, если требуется крутая маршрутизация).
- **UI Библиотека:** Ant Design (antd). Это позволит максимально быстро собрать enterprise-интерфейс, сохраняя строгий ИТ-стиль и не тратя время на ручную верстку базовых компонентов. В Ant Design встроена качественная поддержка Dark Mode, что само по себе даст премиальный вид.
- **Стратегия хакатона (Wow-эффект ++ Простота):** 
  - Используем мощные готовые компоненты (Data Tables, Modal, Steps из antd).
  - Крайне важно для защиты: микро-анимации (например, framer-motion для плавно всплывающего дашборда) и интерактивные графики (например, радар компетенций через Recharts/Ant Design Charts).
  - Внедрение Monaco Editor для окна "Код" — выглядит как настоящая IDE внутри браузера. Это технически не очень сложно, но на защите проекта всегда вызывает ВАУ-эффект.

## Основные Экраны и Задачи (Scope of Work)

Процесс разработки разбит на две крупные ветки, чтобы распараллелить работу.

### 1. Сайт-Дашборд и Внутренние страницы (Основа)
Контентно-образовательная часть портала:
- **Главная страница (Landing/Home):** Презентация продукта для ИТ-специалистов, быстрая статистика, мотивационные блоки.
- **Дашборд (Каталог Курсов):** Сетка карточек с направлениями, бейджами по уровню сложности и фильтрацией. 
- **Внутренняя страница Курса:** Описание, список уроков (собранный через компонент `Timeline` или `Steps` из antd, отображающий прогресс прохождения - зеленая галочка, если `status` == success).
- **Плеер лекций и Интерактивная IDE:** 
  - Сплит-экран. Слева контент лекции, справа — редактор кода (Monaco Editor).
  - Снизу консоль, стилизованная под темный терминал. Студент тестирует код, жмет "Run", и получает ответ от бэкенда через API Gateway.

### 2. Личный кабинет и Авторизация
Персональная часть и геймификация:
- **Авторизация/Регистрация:** 
  - Реализация интерфейса SignIn / SignUp (модальные окна или отдельные роуты).
  - Интеграция с готовым auth-бекендом (`supabase.auth.signInWithPassword`), чтобы не тратить время на парсинг токенов. Все управление сессией остается на SDK.
- **Профиль Студента (Личный Кабинет):**
  - **Матрица компетенций:** Интерактивный паутинный график (Radar chart), перечисляющий прокачку (Python, SQL, DevOps).
  - Базовая сводка по последним решенным заданиям (информация из таблицы `SUBMISSIONS`).
  - Управление своими песочницами (Sandboxes) - список активных поднятых серверов с кнопкой "убить докер" или ссылками на них.

## Timeline реализации (Детальный технический план на Хакатон)

Расчет времени ведется в рамках плотного кодинга на хакатоне. Каждая фаза разбита на конкретные микро-задачи технического уровня, чтобы задачи можно было повесить на доску (Trello/Jira).

### Шаг 1: Подготовка инфраструктуры приложения (Часы 1-4)
*Закладываем базу, чтобы потом не отвлекаться на конфиги.*
- [ ] **Микрозадача 1.1: Инициализация проекта и конфигов.**
  - Выполнить `npx create-vite@latest frontend --template react-ts`.
  - Настроить `tsconfig.json` (включить strict mode, настроить alias paths `@/*` для абсолютных импортов).
  - Установить линтеры: ESLint + Prettier (базовые конфиги без жестких правил, чтобы экономить время).
- [ ] **Микрозадача 1.2: Интеграция Ant Design.**
  - Выполнить `npm i antd @ant-design/icons`.
  - В `App.tsx` обернуть приложение в `ConfigProvider`. Задать `theme` с использованием фирменных цветов NeoFlex (переопределить `colorPrimary`). Обязательно включить `theme.darkAlgorithm` как базовую дизайн-тему.
- [ ] **Микрозадача 1.3: Настроить Роутинг.**
  - Выполнить `npm i react-router-dom`. Создать файл `router.tsx` с `createBrowserRouter`.
  - Разметить пустые компоненты-заглушки для всех страниц: `Landing`, `Dashboard`, `CourseView`, `LessonPlayer`, `Profile`, `Auth`.
  - Создать `RootLayout` c боковым меню навигации (используем `<Layout>` и `<Menu>` из antd).
- [ ] **Микрозадача 1.4: Интеграция слоя данных (Supabase SDK).**
  - Выполнить `npm i @supabase/supabase-js`.
  - Создать синглтон инициализации клиента в `src/api/supabase.ts`.
  - Подключить ключи (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) и настроить чтение переменных окружения из `.env`.

### Шаг 2: Реализация Авторизации и Личного Кабинета (Часы 4-8)
*Реализуем геймификацию и профиль как одну из ключевых wow-фич для защиты.*
- [ ] **Микрозадача 2.1: Формы Auth и State Management.**
  - Установить легковесный стейт-менеджер: `npm i zustand` (быстрее и проще Redux). Создать `useAuthStore.ts` для хранения данных объекта `user` и session.
  - На странице `Auth` сверстать вкладки (Tabs из antd) "Вход" и "Регистрация".
  - Использовать встроенный компонент `<Form>` из antd. Обработать onFinish и пробросить связку данных напрямую в SDK `supabase.auth.signUp()` или `signInWithPassword()`.
- [ ] **Микрозадача 2.2: Роутинг безопасности.**
  - Написать `ProtectedRoute.tsx` компонент-обертку. Обвесить им приватные роуты (Дашборд, Профиль). Логика: если `useAuthStore` пуст -> редиректим `<Navigate to="/auth" />`.
- [ ] **Микрозадача 2.3: Страница "Профиль студента".**
  - Сделать data-loading данных из таблицы `USERS` и вывести в UI-карточку (`<Card>` avatar + ФИО).
  - Сформировать матрицу компетенций (выполнить запрос на Backend `GET /api/users/me/competencies` через axios).
  - Фронт не высчитывает баллы, он только рисует JSON. Отрендерить `RadarChart` компонент (библиотека recharts). Добавить красивую градиентную заливку и анимацию появения.

### Шаг 3: Дашборд и Каталог курсов (Часы 8-14)
*Создаем витрину продукта (интерфейс потребления контента).*
- [ ] **Микрозадача 3.1: Верстка Сетки Курсов (Thin Client).**
  - В процессе разработки фронт использует массив-заглушку (mock data), а в финале запрашивает API `GET /api/courses` (через React Query). Backend сам собирает, фильтрует и отдает готовые JSON.
  - Сверстать отзывчивую сетку через `<Row gutter={16}>` и `<Col>` из antd.
  - Обернуть каждую запись в `<Card hoverable>`, просто расставляя присланные бэком данные по нужным местам (заголовок, картинки, теги).
- [ ] **Микрозадача 3.2: Агрегация Прогресса.**
  - На карточке внедрить компонент `Progress` из antd: `<Progress type="circle" percent={progress_percent}>` (возвращает круговой индикатор выполнения).
  - Это покажет жюри, что система встроена в цикл обучения.
- [ ] **Микрозадача 3.3: Внутренняя страница курса.**
  - Реализовать переход по параметру `:courseId` (`/course/123`).
  - Обратиться к Backend эндпоинту `GET /api/courses/123/lessons`, который сам сджоинит данные базы и вернет готовое дерево лекций.
  - Вывести эту структуру через `<Steps direction="vertical">` от antd. Расставить зеленые чекбоксы на пройденных шагах, просто читая значение `status === 'success'` из присланного JSON.

### Шаг 4: IDE "Wow-фичи" и Плеер (Самое важное для жюри) (Часы 14-22)
*Технически самая комплексная часть фронтенда, обеспечивающая максимум баллов на питче.*
- [ ] **Микрозадача 4.1: Сборка Layout для Плеера.**
  - Сделать скрытие боковой навигации для маршрута `LessonPlayer` (оставить full width).
  - Разделить экран (flexbox): по 50% между блоком теории (выполнить рендер текста через пакет `react-markdown`) и интерактивным блоком практики.
- [ ] **Микрозадача 4.2: Интеграция Редактора Кода.**
  - Выполнить `npm i @monaco-editor/react`. Встроить компонент `<Editor>`.
  - Динамически менять language props редактора (если `task.type` === `'sql'`, то `<Editor language="sql"/>`). Задать кастомную темную тему `theme="vs-dark"`.
  - Разместить большую первичную кнопку "Run/Submit" для отправки ответа.
- [ ] **Микрозадача 4.3: Интеграция Запуска Кода (Связь с Backend Gateway).**
  - Пакетом `npm i axios` написать сервис вызова API `api/gateway.ts`. Метод POST отправки payload'а с `user_code` на Go-сервер.
  - Реализовать "skeleton loader" на время асинхронного ожидания ответа (await). Задизейблить кнопку, повесив `<Button loading>`.
- [ ] **Микрозадача 4.4: Консоль вывода ответов.**
  - Сверстать блок терминала (`<div className="term-window">` с черным фоном и зеленым шрифтом monospace).
  - Вывести в него `console_output` от Backend'а. Встроить логику парсинга статуса: если success — включить аниматор 🎉 (`npm i react-confetti`), чтобы выделиться на презентации.

### Шаг 5: Полировка и "Продающий UX" (Финальные часы)
*То, что превращает сырой код во вкусный коммерческий продукт.*
- [ ] **Микрозадача 5.1: Уведомления об ошибках и таймаутах.**
  - Интегрировать глобальный контекст `App.useApp()` (из antd) для генерации всплывающих ошибок: `message.error("Стенд не отвечает, попробуйте позднее")` или пуш-тост `notification.success()`. Это покажет зрелость решения.
- [ ] **Микрозадача 5.2: Интеграция Sandbox стендов.**
  - Добавить в плеере кнопку "Развернуть Sandbox ВМ" для тасок с типом `sandbox`.
  - По таймауту получать от бэка `container_url`, отображать его в iFrame сбоку (или Modal) или кнопкой "Открыть в новой вкладке" для наглядности изолированного стенда.
- [ ] **Микрозадача 5.3: Микро-анимации страниц.**
  - Внедрить `framer-motion`: обернуть главные компоненты страниц в обертку `initial={{ opacity: 0}} animate={{ opacity: 1 }}`. Это создаст эффект плавных мягких переходов ("дорогих" анимаций) между кликами, что подкупает судейскую комиссию.
