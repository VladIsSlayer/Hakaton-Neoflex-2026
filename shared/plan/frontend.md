# Frontend Specification (React Web Portal)

## Архитектура Клиентской Части и Стилистика
- **Core:** React 18+ поверх сборщика Vite (ил Next.js для SSR, если требуется сложный роутинг).
- **Дизайн Код (КРИТИЧЕСКИ ВАЖНО):** Необходимо избегать скучных "дефолтных" компонентов. Поскольку продукт подается как корпоративное решение для NeoFlex, нужно использовать их фирменный строгий IT-стиль. 
  - Цветовая палитра: контрастная, поддержка качественного **Dark Mode** (темной темы), потому что инженеры чаще работают в dark theme.
  - Компоненты (Кнопки, Input, Cards) должны выглядеть премиально с легкой микро-анимацией (framer-motion или css-transitions).
- **Data-Слой:** Использование `@supabase/supabase-js`. Прямое общение с базой (подписки, CRUD) производится непосредственно с фронтенда с авторизацией через Token. Для вызовов специфичной и тяжелой внутренней логики используется `Axios`, стучащиийся в API Gateway.

## Основные Экраны и Зоны (Портал Студента)

Второе имя портала студента — "Личный кабинет инженера". Это не просто читалка текста, а интерактивный дашборд.

### 1. Каталог потоков и виджет прогресса (Dashboard)
- Сводка открытых курсов с процентами завершенности (`ENROLLMENTS.progress_percent`).
- Вывод "Матрицы ИТ-Компетенций" в профиле. Эта диаграмма показывает радар навыков (Radar Chart - скиллы вроде SQL, DevOps, React) на основе данных из `USER_COMPETENCIES`. Дает понимание того, насколько ученик усвоил материал.

### 2. Плеер и Среда выполнения (IDE/Sandbox View)
Когда студент заходит в урок (`content_type = interactive_task` / `code` / `sql`), интерфейс делится на сплит-зоны (Split Pane Layout).
- **Слева:** Задание, справочный текст или видеоматериал.
- **Справа (Верх): Мульти-Редактор Кода.**
  - Использовать библиотеку-компонент **Monaco Editor** (как в VS Code). Обеспечивает подсветку синтаксиса, нумерацию строк.
  - Кнопка Execute/Run: Посылает payload `user_code` на `API Gateway`. Кнопка блокируется в состояние Loading (скелетон лоадер).
- **Справа (Низ):** Панель "Терминал/Вывод".
  - Стилизовано под консольное окно (черный фон, monospace шрифт).
  - Получает асинхронный ответ от `Code Checker`, красиво рендерит текст успеха или крашлога-ошибки (`SUBMISSIONS.console_output`).

### 3. Работа с песочницами (Sandboxes)
- Если задача предусматривает полноценный стенд Docker, интерфейс рисует кнопку: "Развернуть Sandbox среду".
- При нажатии, отображается визуальный таймер и спиннер. По завершению, когда микросервис выделил ресурсы — на экране появляется `container_url`. Пользователь кликает, открывая отдельную вкладку браузера для работы.

## Основные Экраны (Панель HR и Администратора)

Интерфейс для контент-мейкеров и аналитиков компании.
- **Course Builder (Конструктор треков):** Drag-n-drop (через `dnd-kit` или похожую либу) интерфейс сборки модулей из кусочков (Урок + Тест + Код). Записи курсов и уроков добавляются прямо в БД `COURSES` / `LESSONS`. Удобное прикрепление эталонных ответов к задачам.
- **Аналитика Прогресса (Reports):** Таблица списка обучаемых, где видны метрики от "Analytics & Matrix Service", например, отставание от среднего среза группы.

## Ответственность Frontend Разработчика в решении
1. Сделать интеграцию аутентификации через API (Login/Refresh). Убедиться в корректности JWT.
2. Обрабатывать потенциальные отказы бекенда (если скрипт повис или выполняется бесконечно — обрабатывать таймауты).
3. Придерживаться компонентного подхода под UI Kit (кнопки, статусы с бейджами, модальные окна, таблицы) без "hardcoded" стилей в разметке. Фронт должен выглядеть ВАУ-эффектно для жюри хакатона.
