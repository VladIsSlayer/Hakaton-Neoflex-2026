# Backend Services Stack (Спецификация для Go/FastAPI)

## Технологическая база
- **Язык программирования:** Желательно Golang (так как он идеален для микросервисной легковесной среды) или Python (FastAPI).
- **Развертывание (Деплой):** Docker / Docker-compose сеть (или K8s-манифесты/helm).
- **Связь между сервисами:** Внутренний трафик проходит по REST (или gRPC между собой). Приватные сервисы не "торчат" наружу, пользователь взаимодействует только с Router (Gateway).

## Архитектура Сервисов и Обязанности (Responsibilities)

Проект поделен на 5 узкоспециализированных микросервиса.

### ✨ API Gateway / Router (Entrypoint)
- **Точка входа:** Принимает все вызовы от веб-клиента, которые не идут напрямую в Supabase PostgREST.
- **Проверка Подлинности:** Читает HTTP Bearer Token. Извлекает JWT, валидирует его (подпись Auth) и пробрасывает объект `user_id` в заголовке к приватным микросервисам, блокируя неавторизованные сессии.
- **Маршрутизация:** Например, запросы вида `POST /api/checker/execute` перенаправляет в "Code/SQL Checker Service". 

### 1. Code/SQL Checker Service 
Самый критичный к безопасности микросервис. 
- **Функция:** Получает `user_code` от Студента, загружает из базы эталонный `reference_answer` для этого задания, и выполняет присланный скрипт или SQL запрос.
- **Изоляция:** Код не должен выполняться непосредственно в среде бэкенда! Если это Python/Go код ученика — нужно использовать временные изолированные песочницы (например запуск через `Docker SDK` или хотя бы `os/exec` chroot-нутым). 
Если это SQL запрос — он должен исполняться под урезанным (Read-only) юзером в базе данных (Не администратором!), чтобы предотвратить `DROP TABLE`.
- **Запись:** Возвращает результат stdout (ошибочный трейсбек или success) в таблицу `SUBMISSIONS`.

### 2. Sandbox Manager 
Оркестратор песочниц - выдает ресурсы "по требованию".
- **Интеграция:** Подключается к сокету Docker Daemon внутри хоста (или использует Kubernetes API `client-go`).
- **Сценарий работы:** 
  - Студент запрашивает песочницу (нажать кнопку).
  - Сервис достает `docker_image` из задачи (например, CentOS сервер или базу RabbitMQ). Стартует контейнер со случайным свободным host-портом с ограничением по CPU/памяти.
  - Формирует ссылку/URL на этот сервис (через reverse proxy) и заносит её в `SANDBOXES` таблицу со сроком жизни `expires_at` (например, current_time + 2 часа).
  - Студент получает инстанс для работы изолированно от других.
- **Garbage Collector:** Крон-задача внутри микросервиса, которая раз в минуту ищет "протухшие" контейнеры в таблице БД и посылает команду `docker stop -> docker rm`.

### 3. Analytics & Matrix (Прогресс и компетенции)
Сервис, решающий главную бизнес-боль (мониторинг ИТ-навыков).
- **События:** Можно реализовать как Background worker (слушает trigger или webhook из Supabase, когда в таблице `SUBMISSIONS` появляется статус `success`).
- **Просчет уровня (Матрицы компетенций):** Если ученик успешно сдал задачу из раздела "Kubernetes", сервис добавляет +5% к его навыку Kubernetes в таблице `USER_COMPETENCIES`.
- **Просчет прогрессии:** Перерасчет "опережения/отставания" графика для студента, запись значения прогресса (`progress_percent`) в `ENROLLMENTS`. Дает HR-отделу возможность находить "буксующих" студентов.

### 4. Git/SVN Reviewer Listener
Слушатель событий (Webhooks Server).
- **Интеграция:** Используется для задач на программирование, где требуется реальный Pull Request (на уровне корпоративных стандартов). 
- Сервис выставляет HTTP endpoint (например, `/webhooks/bitbucket`). Когда ментор или CI делает Accept Pull Request в корпоративном репозитории, сервис ловит JSON payload webhook'a, ищет номер задачи в названии ветки или коммита (например, "fix: DEVOPS-77") и закрывает это обучение как _Пройденное_ в таблице `SUBMISSIONS`.

### 5. Интеллектуальный Telegram Bot
Контактная точка (Опциональный сервис).
- **Long Polling / Webhook ТГ:** Слушает сообщения пользователя. 
- Пользователь вводит свой токен/deep-link, привязывает аккаунт TG к профилю в LMS.
- Бот использует push-архитектуру, присылая: "Ваше задание проверено!", "Вам зачислена компетенция Docker". Может отвечать на ЧАВО (FAQ) вопросы для экономии ресурсов поддержки.
