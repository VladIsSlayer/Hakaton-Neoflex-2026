# Frontend Status (React + TypeScript)

## Что уже реализовано

### Технологический стек
- React + TypeScript + Vite.
- UI: Ant Design.
- Роутинг: `react-router-dom`.
- Данные: `@tanstack/react-query`.
- Состояние UI-auth: `zustand`.
- Источник данных на текущем этапе: Supabase (`@supabase/supabase-js`) напрямую из фронта.

### Экраны и UX
- Собраны и связаны основные страницы: `Landing`, `Dashboard`, `Lessons`, `CourseView`, `LessonPlayer`, `Profile`, `Auth`.
- Хедер, навигация и сценарии demo-mode (logged out / logged in) работают.
- Для logged out:
  - курсы и лекции видны,
  - персональный прогресс/баннеры скрыты,
  - переходы с CTA на регистрацию.
- Для logged in:
  - персональные блоки, прогресс и профильные данные подтягиваются.

### Работа с контентом (ключевое изменение)
- Уроки переведены на модульный рендер:
  - `content_blocks_json` с типами `text`, `video`, `quiz`, `ide`.
- Курсовое окно (в т.ч. для тестового Course 4) переведено на такой же принцип:
  - курсовый контент читается из `courses.content_blocks_json`.
- Сохранена обратная совместимость с legacy-полями lessons (fallback), чтобы миграция БД проходила без поломок.

### Практика/IDE
- Для IDE реализованы:
  - выбор языка (`sql/python/go/javascript`),
  - нумерация строк,
  - тест-кейсы из БД,
  - отдельный текст задачи (`ide_task`/`task`).
- Кнопка проверки сейчас формирует JSON payload, показывает `alert` и mock-ответ (заглушка до backend).

### Профиль
- Профиль опирается на данные БД:
  - роль пользователя, `tg_chat_id`,
  - статусы задач,
  - последние решения,
  - расчетные KPI.
- Добавлены пагинации по спискам и выравнивание карточек по высоте.

## Что делаем по-другому (по сравнению с ранним планом)

- Ушли от идеи “жестко захардкоженные страницы + mock как основной режим”.
- Базовая модель теперь data-driven:
  - уроки и курсы рендерятся из JSON-блоков БД.
- Временно используем Supabase direct-read как источник данных для ускорения фронтовой разработки.
- Подготовили фронт под плавный переход к thin-client через backend API (без переписывания UI-компонентов).
